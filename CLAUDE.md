# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## What this repository is

A **document archive**, not a code project. It contains scanned/original PDFs of Texas Instruments documentation for the TMS340 graphics processor family (TMS34010 and TMS34020) and its surrounding toolchain. The stated purpose (`README.md`) is AI-driven analysis of these sources, and over time the scope has been broadened to: original tool disks, emulator/decompiler upstream pointers, hardware-usage indexes, and historical articles.

There is no build system, no tests, no linter, no application code. Do not invent commands. If asked to "run" or "build", clarify — there is nothing to run. The two scripts in `scripts/` (`download_sources.py`, `hash_manifest.py`) are stdlib-only Python and exist to maintain the archive, not to ship runnable software.

## Repo layout (high level)

- `docs/` — TI manuals, datasheets, articles, patents (`ti-official`, `ti-related`, `articles`, `patents`, `datasheets`, `manuals`, `third-party`).
- `tools/` — toolchain manuals + original disk images (`compiler`, `assembler`, `debugger`, `tiga`, `original-disks`, `extracted`). `tools/tiga/promo-kit-scans/` holds the unpacked TIGA promo-kit imagery.
- `software/` — graphics library, TIGA demos, examples.
- `hardware/` — usage of the '34010/'34020 silicon (`arcade/{atari,midway,art-magic,ice-game-room,tch}`, `pc-tiga`, `amiga`, `mac`, `workstation`, `embedded-avionics`, `silicon-die/{tms34010,tms34020,tms34082}/`).
- `emulation/` — upstream pointers to MAME (`emulation/mame/UPSTREAM.md`) and to the community `tms34010-sdk` mirror (`emulation/UPSTREAM_SDK.md` + `tms34010-sdk-README.md`). No full clones by default.
- `bibliography/`, `wanted/`, `scripts/`, `incoming/`.

Indexes live at the repo root: `MANIFEST.csv`, `SOURCES.md`, `WANTED.md`, `LEGAL_NOTES.md`. Topical indexes within the tree: `hardware/arcade/ARCADE_USES.md` (arcade-game → MAME-driver map with verified clock citations).

## Choosing the right document

When the user asks a question, the answer almost always lives in one of these PDFs. Pick by topic:

- **What a pin does, electrical timing, register bit layouts on the '34010** → `docs/datasheets/84292.pdf` (SPVS002C, the '34010 datasheet, June 1991). For the **earlier 1987-07 SPVS002A revision** (sibling, sometimes useful when something seems to have changed between revisions), see `docs/datasheets/SPVS002A_TMS34010_Graphics_System_Processor_198707.pdf`.
- **Architecture / instruction reference for the '34010** → `docs/ti-official/1988_TI_TMS34010_Users_Guide.pdf` (SPVU001A, 1988). For first-silicon vs. revised behavior comparisons, also consult the **1986 first edition** at `docs/ti-official/1986_SPVU001_TMS34010_Users_Guide_first_edition.pdf` (SPVU001, no `A` suffix). Default to the 1988 revision; cite the 1986 edition only when revision diffs matter.
- **Palette / RAMDAC chips that pair with the '34010/'34020** (TLC34074 DAC, TLC34075/TLC34075A palette, TLC34076 palette) → `docs/datasheets/`. The 1992 SLAS054 TLC34076 manual (`1992_SLAS054_TI_TLC34076_Video_Interface_Palette_Data_Manual.pdf`) is the most thorough; the **1995 production-data revisions** (`XLAS056_TLC34074_*`, `XLAS058_TLC34075A_*`, `XLAS076_TLC34076_*`) are concise data-sheet style. Use these for VRAM-to-monitor signal-path questions, color-lookup-table layout, and pixel-clock options. The TIGA-era boards in `hardware/pc-tiga/` typically pair a '34010 with one of these.
- **Anything '34020-specific** (the second-generation part has a wider instruction set, different host interface, etc.) → `docs/ti-official/2564006-9721_TMS34020_Users_Guide_Aug90.pdf`. Do **not** assume '34010 behavior carries over.
- **TMS34061 Video System Controller** (the VRAM/CRTC companion chip — sometimes paired with the '34010 instead of, or alongside, a separate RAMDAC) → `docs/ti-official/TMS34061_Users_Guide.pdf` for the full programming/architecture reference, and `docs/datasheets/SPPS010A_TMS34061_Video_System_Controller_198602.pdf` (Feb 1986) for pinout / electrical / register-summary form.
- **TMS34082 floating-point coprocessor (programmer-level reference)** → `docs/ti-official/TMS34082_Designers_Handbook_1991.pdf`. This is the **engineering** doc for the '34082 — register set, instruction repertoire, host-bus protocol, board-design guidance. Use this **before** reaching for `docs/patents/US5025407.pdf`, which only describes the FPU at claim/architecture-intent level.
- **Instruction encodings, assembler directives, COFF, linker behavior** → `tools/assembler/TMS34010_Assembly_Language_Tools_Users_Guide_SPVU004.pdf`. Quick-lookup form: `tools/assembler/TMS34010_Assembly_Language_Tools_Reference_Card.pdf`.
- **C language extensions, calling convention, runtime** → `tools/compiler/TMS34010_C_Compiler_Reference_Guide_1988.pdf` (default). For the **earlier 1986 edition** (first-silicon / pre-revision-A behavior) see `tools/compiler/1986_TI_TMS34010_C_Compiler_Users_Guide.pdf`. Quick-lookup form: `tools/compiler/TMS34010_C_Compiler_Reference_Card.pdf`.
- **C source-level debugger** (the TI-shipped debugger that goes with the C toolchain) → `tools/debugger/1991_SPVU021A_TMS340_Family_C_Source_Debugger_Users_Guide.pdf` (SPVU021A, Sept 1991).
- **Graphics primitives shipped by TI (line draw, fill, blit wrappers, etc.)** → `software/graphics-library/spvu027.pdf` (SPVU027, Aug 1990 — the **family-wide** library covering '34010 *and* '34020). For the **earlier '34010-only** library that this superseded see `software/graphics-library/1987_TI_TMS34010_Math_Graphics_Function_Library_Users_Guide.pdf`. Quick-lookup form: `software/graphics-library/TMS34010_Math_Graphics_Library_Reference_Card.pdf`. An alternate scan of SPVU027 (different SHA256, same content) exists at `software/graphics-library/SPVU027_TMS340_Graphics_Library_199008_altscan.pdf` — use only for scan-quality comparison.
- **Host-side PC API for TIGA boards** → prefer `tools/tiga/SPVU015C_TIGA_Interface_Users_Guide_199009.pdf` (newer/more complete) over `tools/tiga/1989_TI_TIGA-340_Interface_Users_Guide.pdf`. Consult the 1989 one only when comparing revisions. For TIGA's **icon/logo/branding conventions** (a separate companion document, not API material), see `tools/tiga/SPVU018A_TIGA_Interface_Art_199009.pdf`. Promo-kit imagery is unpacked under `tools/tiga/promo-kit-scans/`.
- **Which third-party board / library existed for the '340 family** → `docs/ti-related/1990-340-Family-THIRD-PARTY-GUIDE-4th-edition.pdf`.
- **The TI-branded eval/dev board hardware** — for the **'34010 SDB**, see `docs/ti-official/1987_TI_TMS34010_Software_Development_Board_Users_Guide.pdf` (long form) or `docs/ti-official/TMS34010_SDB_Pocket_Reference.pdf` (quick-card form). For the **'34020 SDB**, see `docs/ti-official/2558670-9761B_TMS34020_Software_Development_Board_Users_Guide_1991.pdf`. For the TI-branded **PC graphics board** that hosts the '34010 (the TIGA-era reference board), see `hardware/pc-tiga/TI_Color_Graphics_Controller_Board_Users_Guide_1986.pdf`; board photos at `hardware/pc-tiga/34010_devbd.jpg` and `34020_devbd.jpg`.
- **Original TI tool disks** (the actual floppies that shipped the assembler, compiler, simulator, TIGA SDK, GFX library, C source debugger, etc.) → `tools/original-disks/` (.img / .DSK / .imd / .zip per disk). Unpacked archives, where present, are under `tools/extracted/`. These are preservation copies, not directly executable; see `MANIFEST.csv` for sizes and provenance per disk.
- **Arcade games using the '34010 / '34020** (which game, which board, which MAME driver, what clock) → `hardware/arcade/ARCADE_USES.md`. Each row cites the MAME `.cpp` file + line number at the pinned MAME commit recorded in `emulation/mame/UPSTREAM.md`. **Trust this index over generic web sources** for board/clock claims; trust MAME source over this index when they disagree (and update the index).
- **MAME source — CPU core and drivers** → don't grep the repo; read `emulation/mame/UPSTREAM.md`. It records the pinned upstream commit, paths under `src/devices/cpu/tms34010/` for the CPU core, and the driver paths (`midyunit`, `midtunit`, `midxunit`, `midwunit`, `harddriv`, `metalmx`, `artmagic`, `lethalj`, `littlerb`, `coolpool`, `micro3d`, `btoads`) for every '34010/'34020-using driver. No upstream code is mirrored locally.
- **Community SDK mirror (`tms34010guy/tms34010-sdk`)** — what is novel vs. duplicated by bitsavers → `emulation/UPSTREAM_SDK.md`. Pinned to commit `5692b4773328f49010896b7c47ada4f96bea73f8`. Of 74 upstream files: 34 are byte-identical duplicates of bitsavers files already in this archive; 39 are novel (mirrored locally with `redistribution_status: do-not-redistribute` because the upstream has no `LICENSE`); plus the upstream README at `emulation/tms34010-sdk-README.md`.
- **Silicon die-shots ('34010, '34020, '34082B)** → `hardware/silicon-die/{tms34010,tms34020,tms34082}/bercovici/`. Stitched MZ JPEG plus the rendered DokuWiki pages from siliconpr0n.org. Photographer: Antoine Bercovici (SiliconInsider). License: **CC BY-NC 3.0** — attribution required, non-commercial only. `hardware/silicon-die/_index/tag_vendor_ti/` is the source-of-truth tag listing showing only those three TMS340-family parts have community die-shots (no `mcmaster:ti:tms340*` exists).
- **Historical / fan-site / app-note material** → `docs/articles/`. Notable items: `IEEE_Chasing_Pixels_TMS34010_and_VRAM.html` (Jon Peddie, IEEE/Computer Society — **do-not-redistribute**, local mirror only), `geekdot_tiga-programming.html` (community TIGA writeup), `spra402.pdf` (a TI app note), `1992_Guttag_Interview_about_the_9918.pdf` (lineage context — Karl Guttag's earlier '9918, predecessor to the '34010 he co-architected), the Endless Skye fan-site capture (`34010_endlessskye_*`), and Eric Theunissen's `tms34020_uav_nl_avionics.html` + sidecar `tms34020_uav_nl_avionics_files/` directory of embedded images (TMS34020 in F-16 ADP and ISD avionics displays).
- **TMS34082 floating-point coprocessor / matrix-math claims and architecture** → `docs/patents/US5025407.pdf` (US Patent 5,025,407, Gulley & Van Aken, TI, 1991). This is a *patent*, not a datasheet — language is legal/claim-style, not engineering reference. Use for prior-art questions, IP scope, or to understand the FPU's design intent; don't quote it as if it were a programmer's reference.
- **Foundational '34010 GPU architecture (block diagram, raster ops, selectable pitch)** → `docs/patents/US4718024.pdf` (US 4,718,024, Asal/Guttag/Novak / TI, filed Nov 1985, issued Jan 1988). Fig. 2 shows the canonical TMS34010 block diagram. The same group filed 13 sibling applications in late 1985 / early 1986 covering the rest of the architecture; two of those issued as **US 5,333,261** (X/Y coordinate instruction; `docs/patents/US5333261.pdf`) and **US 5,437,011** (graphics computer system; `docs/patents/US5437011.pdf`). For prior-art / claim-scope questions about the '34010 itself, start here, not at US 5,025,407.
- **VRAM (the storage technology underneath the '34010)** → `docs/patents/US4747081.pdf` (US 4,747,081, Heilveil/Van Aken/Guttag/Redwine/Pinkham/Novak, filed 1983, issued 1988) and its companion `docs/patents/US4663735.pdf` (US 4,663,735, Novak/Guttag, same filing date, issued 1987). These are TI's foundational dual-port video DRAM patents; the TMS340 was designed to drive this memory.

## Known gotchas

- **`software/graphics-library/spvu027.pdf` and `software/graphics-library/TI_TMS340_Family_Graphics_Library.pdf` are byte-identical duplicates** (same SHA256 `b7cba156…`, MD5 `141d99f4…`). Read either, but cite `spvu027.pdf` since that filename matches the TI literature number. The duplicate is flagged in `MANIFEST.csv`; do not delete without explicit user approval. Note that `software/graphics-library/SPVU027_TMS340_Graphics_Library_199008_altscan.pdf` is a **different SHA256** — it is the same SPVU027 document but a separate scan from the `tms34010-sdk` mirror; it is **not** a duplicate of the bitsavers copy.
- **'34010 vs '34020**: they share a family but the '34020 adds instructions and changes parts of the host interface. Always check which chip the user means before quoting register or opcode details.
- **'34010 vs '34061 vs '34082**: these are three different parts. **'34010** is the GPU/CPU. **'34061** is the Video System Controller (memory/VRAM/CRTC controller — sits between the '34010 and the frame buffer). **'34082** is the floating-point matrix coprocessor (companion to the '34020). Don't merge their docs in answers.
- **Two TIGA editions exist** (1989 and SPVU015C/Sept 1990). When citing TIGA APIs, default to SPVU015C. The separate `SPVU018A_TIGA_Interface_Art_199009.pdf` is **not** another API edition — it covers TIGA's logo/iconography conventions only.
- **Two SDB user's guides exist** — one for the '34010 SDB (1987) and one for the '34020 SDB (1991, doc 2558670-9761B). They are *not* revisions of each other; they document physically different boards.
- **`docs/patents/US5025407.pdf` covers the FPU coprocessor, not the '34010 GPU.** The patent explicitly distinguishes "graphics processor" (separate block) from "floating-point coprocessor" (the claimed invention). If a user asks for "the TMS34010 patent" and points at this file, gently correct — this is the '34082-class FPU patent. The foundational '34010 architecture patent is **`docs/patents/US4718024.pdf`** (Asal/Guttag/Novak, filed Nov 1985, issued Jan 1988). See the Patents section in `README.md` for the full set.
- **Use the FPU *manual* before the FPU *patent*** for engineering questions — `docs/ti-official/TMS34082_Designers_Handbook_1991.pdf` is the programmer's reference. `US5025407.pdf` is claim-language and only useful for IP scope / design-intent context.
- **Silicon die-shot images are CC BY-NC 3.0** (Bercovici / SiliconInsider via siliconpr0n.org). Attribution is required for any reuse, and they are **non-commercial-only**. The local copies under `hardware/silicon-die/` are private archival mirrors; do not republish without preserving attribution and the NC clause.
- **`docs/articles/IEEE_Chasing_Pixels_TMS34010_and_VRAM.html` is do-not-redistribute** (IEEE / Computer Society copyright). Read it locally; do not re-host. Most other items in `docs/articles/` are author-copyright archival mirrors — check `MANIFEST.csv` for the redistribution flag before quoting more than fair-use snippets.
- **The MAME tree is *not* mirrored here.** If a question requires reading a MAME driver, fetch it from the upstream commit pinned in `emulation/mame/UPSTREAM.md`; do not assume code is on disk. `hardware/arcade/ARCADE_USES.md` cites line numbers at that pinned commit — re-verify if you bump the pin.
- **Files mirrored from `tms34010-sdk`** carry `redistribution_status: do-not-redistribute` because that upstream has no `LICENSE` file. They are kept locally for personal preservation/research. See `emulation/UPSTREAM_SDK.md` for the dedupe table.
- The PDFs are **large** (the '34020 guide is ~63 MB, third-party guide ~37 MB, '34010 user's guide ~33 MB, TIGA Interface Art ~28 MB, '34082 designer's handbook ~27 MB, assembly tools ~24 MB, '34061 user's guide ~20 MB). Always pass a `pages` range to the Read tool — opening one without paging will fail or waste context. The TI literature numbers and dates above are confirmed from the title pages; trust them when planning which page range to fetch.

## Working in this repo

- Edits will most often be to `README.md`, this file, `MANIFEST.csv`, or markdown indexes (`SOURCES.md`, `WANTED.md`, hardware/arcade lists).
- New material lands under the directory layout above. Do not put new files in the repo root unless they are repo-wide indexes.
- If the user wants OCR'd text, indexes, or extracted notes from these PDFs, those are new artifacts — put them under a sibling like `bibliography/` or alongside the source PDF, and update `MANIFEST.csv` accordingly.
- Git history early on used one-PDF-per-commit messages like `Create <filename>.pdf`. Newer commits group changes by intent (structure, manifest, hardware index). Match the style of recent commits.
- When adding new files, run `python3 scripts/hash_manifest.py --write` to refresh SHA256/size, and add a row to `MANIFEST.csv` with at minimum: title, device, source_url, local_path, copyright_status, redistribution_status.
